IBTrACS data

In this lecture we use data from the International Best Track Archive for Climate Stewardship or IBTrACS for short.

https://www.ncei.noaa.gov/products/international-best-track-archive

IBTrACS is the most complete global collection of tropical cyclones available, and it is developed collaboratively with all the World Meteorological Organization (WMO) Regional Specialized Meteorological Centers, as well as other organizations and individuals from around the world.

Downloading Shapefiles

IBTrACS data sets come is various formats. We’ll use their data sets in shapefile format, available at:

https://www.ncei.noaa.gov/data/international-best-track-archive-for-climate-stewardship-ibtracs/v04r00/access/shapefile/

For illustration purposes we’ll use data from the North Atlantic. There are 2 shapefiles depending on what kind of geometric features you want to work with:

  • points: IBTrACS.NA.list.v04r01.points.zip

  • lines: IBTrACS.NA.list.v04r01.lines.zip

To be more precise, we’ll work with the points geometry shapefile:

https://www.ncei.noaa.gov/data/international-best-track-archive-for-climate-stewardship-ibtracs/v04r00/access/shapefile/IBTrACS.NA.list.v04r00.points.zip

Download Shapefiles

Download a copy of the lines zip-file to your working directory, and decompress it.

Import Shapefile

Assuming that the shapefile is in your working directory (where you have this qmd file) and it has been decompressed, to import the data in R you use the "sf" function st_read():

ibtracs = st_read("IBTrACS.NA.list.v04r00.points")

class(ibtracs)

It turns out that ibtracs is an sf object, and also a data.frame. This means we can manipulate it like any other data table in R (e.g. with dplyr commands). For example, the first 15 attributes (columns) are:

# a few column names (of attributes)
names(ibtracs)[1:15]

[11] "WMO_WIND"   "WMO_PRES"   "WMO_AGENCY" "TRACK_TYPE" "DIST2LAND"

Maps with "ggplot2"

Like we did in the previous lectures, let’s see how to make plots with "ggplot2", "sf", and "rnaturalearth".

# map of countries, as an "sf" object
world_map = ne_countries(returnclass = "sf")

Map of Storms in 1975 (1st attempt)

To make a map of storms in 1975, we first need to filter this data set:

# focus on year 1975
ibtracs_1975 = ibtracs |>
  filter(SEASON == 1975)

Then we can make a plot:

ggplot() +
  # country polygons
  geom_sf(data = world_map) +
  coord_sf(xlim = c(-110, 0),
           ylim = c(10, 60)) +
  # locations of tropical cyclones
  geom_point(data = ibtracs_1975,
             aes(x = LON, y = LAT))

Notice that IBTrACS data has many more tropical cyclones than those in storms. Why? Because IBTrACS contains systems of low wind speed (i.e. baby storms that never became big enough to be named).

Map of Storms in 1975 (2nd attempt)

An interesting feature of the "sf" object ibtracs is that we can also pass this to geom_sf(). As you can tell, geom_sf() understands that ibtracs_1975 is a data set in standard vector format. This means that we don’t need to specify anything for x and y axes. Isn’t that cool?

ggplot() +
  # country polygons
  geom_sf(data = world_map) +
  # locations of tropical cyclones
  geom_sf(data = ibtracs_1975) +
  coord_sf(xlim = c(-110, 0), ylim = c(0, 60))

Map of Storms in 1975 (3rd attempt)

Here’s yet another map, this time coloring storms in 1975 by NAME

ggplot() +
  geom_sf(data = world_map) +
  coord_sf(xlim = c(-110, 0),
           ylim = c(10, 60)) +
  geom_point(data = ibtracs_1975,
             aes(x = LON, y = LAT, color = NAME)) +
  theme(legend.position = "none")

Maps with "leaflet"

We can also use the data ibrtacs_1975, (derived from the shapefile) to create a web interactive map:

ibtracs_1975 |> 
  leaflet() |> 
  addTiles() |> 
  addCircles(lng = ~LON,
             lat = ~LAT)

Color coding tropical cyclones

To add colors, we need to do some data manipulation:

# auxiliary table of colors for each storm
colors1975 = ibtracs_1975 |>
  count(NAME) |> 
  mutate(COLOR = rainbow(n()))

ibtracs_1975 = st_join(x = ibtracs_1975, colors1975)
ibtracs_1975 |> 
  leaflet() |> 
  addTiles() |> 
  addCircles(lng = ~LON,
             lat = ~LAT,
             color = ~COLOR,
             label = ~NAME.x)
